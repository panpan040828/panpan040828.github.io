CSS选择器总结

发表于 2016-03-29   |   分类于 前端笔记   |  

最近在做百度前端公开课第一阶段的任务，第一阶段主要是针对HTML和CSS来进行训练的，然后发现有一些不常用CSS选择器都记得不太清楚，于是就总结了这篇博客，常用的那些就没用写进来。

1. * 通配符
* { margin: 0; } 

#id *{ margin: 0; }
2. X:link X:visited X:hover X:active

这个主要是要注意写的顺序，原因是css后面的优先级比前面的高。

3. X > Y 子元素选择器

与X Y不同的是，X>Y只选择X的下一代为Y的子元素，而不包括其他的子孙元素。

4. X+Y 相邻选择器

匹配X后面的第一个元素，只匹配一个元素。

5. X~Y 相邻选择器

与4不同的是，匹配的是X相邻的所有Y元素

6. X[title] 属性选择器

选择带有title属性的所有的X

X[title=”a”]选择带有title=”a”的所有X

X[title*=”h”]匹配属性值包含h的所有的X

X[title^=”a”]匹配属性值以a开头的属性

X[href$=”.png”]匹配结尾带有特定字符串的X

7. X:checked

匹配带有checked属性的元素

8. X:after
.clearfix:after { 
content: ""; 
display: block; 
clear: both; 
visibility: hidden; 
font-size: 0; 
height: 0; 
} 

.clearfix { 
*display: inline-block; 
_height: 1%; 
}
9. X:not(selector) 否定伪类选择器
10. p::first-line p::first-letter
11. X:nth-child(n)

匹配第n个子元素，从第一个子元素开始索引

X:nth-last-child(n) 从最后一个子元素开始索引

12. X:nth-of-type(n)

X:nth-last-of-type(n)

X:first-child

X:last-child

X:only-child

X:only-of-type

X:first-of-type
