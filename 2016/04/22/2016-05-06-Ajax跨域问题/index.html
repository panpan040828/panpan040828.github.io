一. Ajax跨域问题

跨域：只要协议、域名、端口有一个不同，就被当做不同的域。

ajax跨域：利用ajax技术请求不同域名上的数据。

解决方法：

Web页面上调用js文件时则不受是否跨域的影响（不仅如此，我们还发现凡是拥有”src”这个属性的标签都拥有跨域的能力，比如<script>、<img>、<iframe>）

1. 利用<script>标签的src

<script>标签的src属性并不被同源策略所约束，所以可以获取任何服务器上脚本并执行。

`<script type="text/javascript">
    function addScriptTag(src) {
         var script = document.createElement('script');
         script.setAttribute("type", "text/javascript");
         script.src = src;
         document.body.appendChild(script);
     `}`

    //调用加载ScrptTag函数

    //搜索apple，将自定义的回调函数名result传入callback参数中             

    addScriptTag("http://ajax.googleapis.com/ajax/services/search/web?v=1.0&q=apple&callback=result");
     }

//自定义的回调函数result

             //我们就简单的获取apple搜索结果的第一条记录中url数据    
             alert(data.responseData.results[0].unescapedUrl);

2. 将data类型变为jsonp

  • json是一种数据交换格式

  • Jsonp是一种非官方跨域数据交互协议

    JSON是地下党们用来书写和交换情报的“暗号”，而JSONP则是把用暗号书写的情报传递给自己同志时使用的接头方式。看到没？一个是描述信息的格式，一个是信息传递双方约定的方法

Jsonp的主要思想是：在远程服务器上设法把数据装进js格式的文件里，供客户端调用和进一步处理

Jsonp一个要点就是允许用户传递一个callback参数给服务端，服务端会返回一个callback回调函数。callback是以返回的json数据作为参数

eg: callback({“name”:”xiaoming”})

jsonp包括两个部分：回调函数、数据

回调函数是接收到数据之后执行的函数，回调函数是在请求中指定的

数据作为回调函数的参数

jsonp也是利用script的src标签

在跨域的url请求中指定回调函数

http://域名/json/?callback=handle

二. 用jQuery Ajax代替iframe

1. iframe的缺点

<iframe src="/picture.jpg"></iframe>
  • 搜索引擎的“爬虫”程序无法解读这种页面，搜索引擎只找得到框架，却找不到链接，不利于搜索引擎的优化。
  • iframe的自适应高度的问题
  • 很多移动设备无法显示框架，设备兼容性差

2. 用jQuery Ajax代替iframe

<div id="content" style="border: 1px solid red;width: 210px"></div>
<button onclick="turnPage('weixin.html')">微信</button>
<button onclick="turnPage('tongxunlu.html')">通讯录</button>
<button onclick="turnPage('pengyouquan.html')">朋友圈</button>
<button onclick="turnPage('wo.html')">我</button>

function turnPage(url){
$.ajax({
  type:"post",
  url:url,
  success:function(html){
    $("#content").html(html);
  }
})

}
